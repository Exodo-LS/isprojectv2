What is S.O.L.I.D Object-Oriented Design?

To keep it brief, S.O.L.I.D Object-Oriented Design is a means for programmers and developers to develop simple, robust, and updatable code. S.O.L.I.D breaks down to the following principles.

  1. Single Responsibility Principle
  2. Open-Closed Principle
  3. Liskov Substitution Principle
  4. Interface Segregation Principle
  5. Dependency Inversion Principle

Let's focus on the Single Responsibility Principle since it is the most relatable to the code. At one point in the calculator's development, both calculating the input and managing the history of all the calculations. This was making the Calculate Class do more than what it was supposed to do. That was when a new directory called history was created. The calculations.py file in that directory would contain a class meant for only the history of the calculations. That implied that original calc_test.py would lose its history testing methods and just focus only on doing the calculations. As a result, history_test.py would be the file that would test all the methods relating the history of calculations.



The images here show that each class is only doing one thing. Calculate only handles calculations and history/calculations.py handles the items in, added, or removed from the history of those calculations.
