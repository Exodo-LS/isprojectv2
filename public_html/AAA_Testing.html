Welcome to the AAA Testing Page

This page will briefly explain how AAA Testing works.

What is AAA Testing?

Testing validates that your functions are working as they are expected. AAA Testing is a form of testing that contains the following:

1. Arrange
- This step sets up you test case. Making sure that you have all required imports, databases, etc. for the test
2. Act
- This step focuses what you are actually testing. This may be functions that you are validating from another class using inputs in the testing class
3. Assert
- This is the checking step. The assert statement serves that your input generates the correct output. It will prompt an error if the test passes and if you do not an error, the test passed.

Example Time:

Sometimes a picture does a much better job at explaining definitions. So I have provided an AAA Test from my calculator project.

Let me break down what you see. The test_calc_add function is the function I chose to represent how AAA Testing can be implement.

  1. The arrange step begins where I set up the variable my_tuple with a bunch of floats which represent the values to be added together. I set up what I am planning to add together before I move on to Act
  2. The act step is where I run Calculate.add_numbers(my_tuple). This step calls the add_numbers function from the Calculate class. This function requires a parameter and in the previous step, I set up my_tuple to fulfill the parameter requirement.
  3. Lastly, we approach the assert step. As you can see, there are two assert statements. The first assert checks that Calculate.add_numbers(my_tuple) uses the Addition class. The second assert checks that the addition results in 20.0 through Calculate.get_last().

Hopefully with this example, you have understood and are able to implement AAA Testing when you code your projects.
